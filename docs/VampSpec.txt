# NIGHTGLASS 2026 GLOW-UP KIT

**Codename:** Nightglass  
**Purpose:** Turn a promising urban action game into a dense, systemic, replayable immersive sandbox without requiring blockbuster headcount.

This package is intentionally built around the hardest part of the problem: the connective tissue. It supplies an executable reference simulation for a city that remembers, a data-driven opportunity director, witness-and-rumor propagation, player-style inference, content schemas, validators, balance tooling, portable shader functions, technical textures, audio cues, engine adapter stubs, and a production specification that an agent team can execute.

The design thesis is simple:

> The player should never merely spend power. Every use of power should solve one problem, create a signature, alter somebody's model of the player, and bend the next situation.

That thesis produces the “hidden game”: a city-pressure instrument in which force, stealth, influence, mobility, and systems play are not classes selected on a menu. They are habits the world notices and answers. The player authors a style; the city composes variations around it.

## What is here

- `spec/` — the complete game, systems, presentation, production, and agent implementation specification.
- `reference/nightglass_sim/` — executable Python reference implementation of the systemic core.
- `reference/tests/` — deterministic unit tests.
- `content/` — JSON schemas, tuning, factions, districts, abilities, NPC archetypes, and opportunity templates.
- `tools/` — repository probe, content validator, balance simulator, and report generator.
- `shaders/` — portable HLSL technical-art includes for wetness, layered normals, emissive response, and perception visualization.
- `assets/` — generated LUT, blue-noise/dither/ripple textures, vector HUD glyphs, and restrained UI audio cues.
- `adapters/` — Unity, Godot, and Unreal integration seams. Select only the adapter matching the real repository.
- `.github/workflows/` — validation and deterministic simulation checks.
- `AGENTS.md` — strict execution contract for coding agents.

## First five commands

From this folder:

```bash
python tools/repo_probe.py /absolute/path/to/the/game --out reports/repo_audit
python tools/validate_content.py content
python -m unittest discover -s reference/tests -v
python tools/run_balance_sims.py --runs 500 --turns 36 --out reports/balance
python tools/make_agent_backlog.py reports/repo_audit/report.json --out reports/agent_backlog.md
The repository probe does not modify the game. It detects engine, code and asset topology, likely gameplay systems, duplicate or oversized assets, rendering hooks, physics hooks, test coverage, and integration risk. Its output is the binding layer between this kit and the actual project.

Non-negotiable integration order
Establish a clean boot, deterministic test scene, source-control hygiene, and performance baseline.

Fix movement, camera, interaction latency, hit reaction, and input buffering before adding content.

Add event telemetry and a single vertical-slice district.

Lock the 60 Hz/60 fps feel budget and content memory budget.

Only then expand factions, districts, abilities, vehicles, and narrative arcs.

Scope doctrine
Nightglass is not a demand to build “more open world.” It is a demand to make the existing world denser in causality. A three-block district with twenty meaningful stateful actors is more valuable than ten square kilometers of decorative pavement.

The kit uses original terminology and implementation. It takes inspiration from urban immersive sims, social sandboxes, systemic stealth games, and kinetic action games without copying protected characters, plots, art, dialogue, maps, or proprietary mechanics.

write("spec/00_VISION_AND_GAME_DESIGN.md", r"""

00 — VISION AND GAME DESIGN
The one-sentence game
A dense urban action immersive sim in which the player builds a personal operating style, and a living city learns that style through witnesses, evidence, favors, fear, and desire.

The promise
The player enters each situation with imperfect information and several legible possibilities. They can improvise with movement, violence, stealth, conversation, relationships, environment, and special powers. The immediate action feels excellent. The aftermath matters. The city does not merely raise enemy health; it changes who is prepared, who is afraid, who offers work, which routes remain safe, what rumors circulate, and what the player now owes.

The fantasy is not omnipotence. It is authorship under pressure.

The hidden game
Most half-finished urban sandboxes contain isolated good things: a combat prototype, a dialogue system, a city scene, a few enemies, perhaps vehicles or powers. What they lack is a reason for those things to talk to one another.

Nightglass turns the existing game into a four-layer instrument:

Layer	Player question	System response
Contact	“Does this feel good right now?”	movement, camera, impact, sound, readability
Tactic	“What is my best move in this situation?”	spatial options, enemy roles, resources, social leverage
Consequence	“What will this cost or unlock?”	witnesses, evidence, heat, injury, debt, faction belief
Arc	“Who am I becoming in this city?”	organic playstyle, relationships, district state, legacy
No layer can substitute for another. Systemic depth cannot rescue poor input feel. Spectacle cannot rescue consequence-free missions. Narrative cannot rescue illegible rules.

Five design pillars
1. Power leaves residue
Every strong verb has at least one residue channel:

Force creates sound, injury, fear, evidence, damaged access, and retaliation.

Stealth consumes time, information, patience, routes, and sometimes allies’ trust.

Influence spends leverage, creates promises, exposes motives, and transfers obligation.

Mobility creates visibility, pursuit risk, route knowledge, and positional opportunity.

Systems play consumes tools, access, preparation, and leaves technical traces.

Extraordinary powers increase need, anomaly, instability, or attention.

Residue is not a scolding morality meter. It is material for the next scene.

2. Style is practiced, not selected
The game tracks how the player actually solves problems. It forms a soft style vector across five channels:

FORCE — direct control through damage, threat, armor, disruption
STEALTH — concealment, misdirection, timing, silent access
INFLUENCE — persuasion, favors, status, blackmail, empathy
MOBILITY — pursuit, escape, verticality, vehicles, positioning
SYSTEMS — surveillance, devices, traps, environment, logistics
Equipment, contacts, abilities, and safehouse facilities amplify the style; they do not imprison the player in a class. Unlocks are sidegrades and grammar expansion, not raw percentage inflation.

A player may become a Ghost Broker, Iron Diplomat, Pursuit Artist, Saboteur, Predator, Guardian, or something the design team did not name. The system should describe their style after the fact rather than prescribing it before play.

3. Failure changes the world
Ordinary failure branches into playable cost:

The target escapes and becomes informed.

A witness survives with a partial description.

The player succeeds but suffers an injury that changes handling.

A faction completes a rival objective.

Evidence raises heat for one identity in one district.

A promised favor becomes due.

A route, contact, or safehouse becomes temporarily compromised.

The reward becomes leverage rather than money.

A public success creates fear but reduces trust.

An ally intervenes and now owns part of the outcome.

Reload remains available, but the game should make continuation tempting.

4. The city remembers locally
Global meters are crude. Memory should have location, identity, source, and confidence.

A patrol in Dockglass should not instantly know what a bartender saw in Saint Orra. Rumors propagate through relationships and institutions. Claims distort and decay. Cameras generate different evidence from eyewitnesses. Disguises and vehicles create identity ambiguity. A faction can fear the player without knowing their face. Another can know the face but misunderstand the motive.

This creates useful uncertainty and supports multiple identities, fronts, disguises, and reputation repair.

5. Density beats acreage
Do not compete with the largest map. Compete with the richest block.

A district earns its cost when it supports:

at least three traversal layers,

at least three social strata or access modes,

two or more faction interests,

multiple time/state variants,

reusable systemic spaces,

persistent consequences,

recognizable material and audio identity,

and authored secrets that reward close attention.

The player should learn a place, not merely pass through it.

Fun at every temporal scale
Every 100–250 milliseconds
Inputs are buffered and acknowledged.

Camera response is stable and predictable.

Foot placement, acceleration, aim, and turn response agree.

Interaction candidates are visible before the button press.

Hits produce synchronized pose, sound, VFX, and impulse.

Animation never steals control without a known rule.

Every 3–8 seconds
The player reads a threat, route, tell, resource, or opportunity.

An action changes local state.

Feedback explains the change without requiring a menu.

At least one follow-up choice is opened or closed.

Every 30–90 seconds
The encounter changes phase: suspicion, contact, pursuit, negotiation, search, escape, cleanup.

A resource or relationship shifts.

The player’s method becomes visible to the world.

Every 8–20 minutes
A job or opportunity reaches a meaningful resolution.

The result changes access, pressure, ownership, information, or obligation.

The player receives a new possibility, not only currency.

Every 1–3 hours
A faction agenda advances.

A district changes state.

A relationship crosses a threshold.

A playstyle expression becomes newly possible.

A prior consequence returns in altered form.

The core loop
READ THE CITY
  observe people, routes, tensions, needs, evidence, schedules
        ↓
FORM AN INTENT
  choose target, identity, allies, equipment, promises, acceptable residue
        ↓
ENTER A SITUATION
  travel, infiltrate, converse, pursue, fight, manipulate systems
        ↓
IMPROVISE UNDER PRESSURE
  combine verbs; accept, redirect, or clean up consequences
        ↓
RESOLVE AND INTERPRET
  immediate reward + witnesses + evidence + injuries + beliefs + debts
        ↓
RESHAPE THE NETWORK
  invest in contacts, fronts, safehouses, abilities, district influence
        ↺
Player resources
Avoid a dashboard of twelve bars. Expose four primary concepts and keep detailed state underneath.

Condition
Physical ability to continue: health, injury, poise, stamina, fatigue. Condition answers “can my body execute?”

Need
Pressure that powers or destabilizes extraordinary actions. In a supernatural game it can be hunger; in a crime thriller it can be adrenaline, addiction, compulsion, or neural load. Need answers “what does power demand?”

Mask
How coherent and safe the current identity remains. Mask includes recognition, forensic linkage, disguise integrity, vehicle linkage, and public narrative. It answers “who does the city think did this?”

Leverage
Actionable social power: favors, secrets, promises, access, credibility, status, and witnesses willing to speak. Leverage answers “who can be moved, and at what relational cost?”

Money remains useful but deliberately incomplete. Money buys goods. Leverage changes rules.

Progression
Progression has four orthogonal tracks:

Practice: new expressive options within frequently used methods.

Infrastructure: safehouses, fronts, routes, workshops, surveillance, legal cover.

Relationships: contacts, allies, rivals, dependents, handlers, protégés.

Knowledge: names, schedules, vulnerabilities, rituals, passwords, habits, maps.

Raw damage and health increases are bounded. The late-game character is more capable because they can compose more verbs and control more context, not because earlier enemies become irrelevant.

Practice unlock rules
Track meaningful method use, not button spam.

Award practice when an action resolves risk, creates a new state, or demonstrates mastery.

Use diminishing returns within a short window.

Unlock choices at thresholds; do not auto-spend.

Offer one option that deepens the dominant style, one that hybridizes it, and one that repairs a weakness.

Never silently adapt enemies to nullify the unlock.

Organic archetypes
These are examples for content authors, not hard classes.

Archetype	Signature	Enjoys	Natural cost
Ghost Broker	stealth + influence	unseen access, secrets, proxies	slow setup, fragile when identified
Iron Diplomat	force + influence	coercive presence, bodyguard play	fear outruns trust
Pursuit Artist	mobility + force	chase control, vehicle takedowns	public exposure, collateral
Systems Weaver	systems + stealth	cameras, traps, remote access	dependency on preparation
Street Saint	influence + mobility	rescue, escort, community routes	obligations accumulate
Volatile Adept	force + extraordinary power	explosive tempo shifts	need and anomaly spike
Clean Operator	stealth + systems + discipline	low-residue completions	fewer spectacular shortcuts
Difficulty
Difficulty is a vector, not one multiplier.

Offer presets plus independent controls for:

enemy perception,

enemy coordination,

player damage received,

resource scarcity,

social information clarity,

timing leniency,

pursuit persistence,

consequence severity,

navigation assistance,

and simulation volatility.

Higher difficulty should increase the quality of enemy decisions and reduce informational certainty before it inflates health. Never hide core rules to manufacture difficulty.

Content scale for a premium indie target
A credible high-quality target is not an imitation of a 1,000-person production. Use a dense scope:

3 major districts plus 1 endgame/composite space,

5–7 factions,

8 authored faction arcs,

24 systemic opportunity templates,

12 high-specificity signature missions,

20–28 combat/social NPC archetypes,

5 player method families with 8–12 meaningful sidegrades each,

12–18 vehicles only if driving is already core,

40–60 named relationship characters,

150–250 reusable barks per faction distributed through compositional grammar,

20–35 hours for a first campaign,

a materially different 12–20 hour replay,

and an open-ended city-pressure mode after resolution.

Cut acreage, filler collectibles, duplicate weapons, and noninteractive interiors before cutting systemic density.

The anti-feature list
Do not add:

map-icon confetti,

randomized loot tiers that obsolete authored equipment,

bullet-sponge elites,

universal psychic police,

a single global morality bar,

dialogue choices whose outcomes are merely cosmetic,

crafting that exists only to convert five junk currencies into one useful item,

mandatory daily chores,

procedural missions with no authored grammar,

cinematic animation that makes controls feel underwater,

or a “next-gen” post-process stack that destroys readability and temporal stability.

The north star is not quantity. It is consequence with grace.
""")

write("spec/01_SYSTEMS_SPEC.md", r"""

01 — SYSTEMS SPECIFICATION
1. Gameplay event spine
The entire glow-up depends on one semantic event stream. Existing systems remain authoritative for their domains; the event spine reports meaningful state transitions.

Canonical event
{
  "event_id": "evt_0000123",
  "tick": 91220,
  "kind": "guard_incapacitated",
  "actor_id": "player",
  "target_ids": ["npc_guard_14"],
  "district_id": "dockglass",
  "intensity": 0.72,
  "visibility": 0.45,
  "method": "stealth",
  "identity_key": "id_night_courier",
  "tags": ["nonlethal", "restricted_area", "body_hidden"],
  "metadata": {
    "noise_db_proxy": 0.12,
    "camera_coverage": 0.0,
    "weapon_id": "wire_01"
  }
}
Event requirements
IDs are unique and stable.

tick is simulation time, not frame count or wall-clock time.

kind describes a completed semantic fact, not an input.

intensity is normalized and has event-specific interpretation documented in the registry.

visibility estimates how observable the event is before witness traits and occlusion.

method describes the dominant resolution method.

identity_key may be unknown, partial, disguised, vehicle-linked, or true.

Metadata is bounded. High-volume transforms do not belong in this stream.

Events may be replayed deterministically.

Consumers are idempotent by event_id.

Initial event registry
Start with fewer than forty event kinds:

entered_restricted_area, exited_restricted_area
object_stolen, object_planted, evidence_destroyed, evidence_created
npc_threatened, npc_persuaded, promise_made, promise_broken
favor_called, favor_completed, secret_learned, secret_exposed
attack_started, hit_landed, npc_incapacitated, npc_killed
player_injured, player_spotted, identity_compromised
alarm_raised, alarm_silenced, search_started, search_ended
pursuit_started, pursuit_escalated, pursuit_escaped
vehicle_stolen, vehicle_damaged, vehicle_abandoned
system_hacked, power_cut, door_breached, route_opened
objective_acquired, objective_lost, job_resolved, job_abandoned
district_asset_changed, faction_asset_changed, safehouse_compromised
2. Pressure model
Pressure converts residue into future play. It is not a punishment layer; it is a scenario generator.

Pressure channels
Channel	Scope	Generated by	Relieved by	Creates
Exposure	identity + district + faction	witnesses, cameras, evidence, public powers	cleanup, false narrative, disguise retirement, favors	prepared enemies, recognition, access denial
Heat	district/institution	alarms, violence, theft, pursuit	time, bribery, diversion, rival crisis	patrol density, checkpoints, searches
Need	player	power use, injury, stress, abstinence	feeding/rest/ritual/medication depending fiction	instability, tempting shortcuts
Injury	body	damage, overexertion, crash	treatment, rest, adaptation	altered handling, vulnerability, scars
Debt	relationship	intervention, promises, borrowed access	service, payment, renegotiation, betrayal	calls, constraints, alliance arcs
Anomaly	world/faction	impossible events, forbidden tools	suppression, disinformation, containment	specialist hunters, research, cult interest
Volatility	district	unresolved conflict, scarcity, leadership loss	stabilization or faction victory	emergent opportunities, danger, price shifts
Pressure rules
Pressure must be attributable. The UI can answer “why did this rise?”

Pressure decays at different rates and only under plausible conditions.

A high channel opens content before it closes content.

At least two relief methods exist for each channel.

Relief methods leave their own residue.

Pressure is local by default; global only when fiction supports it.

Pressure thresholds are soft bands with hysteresis, not flickering exact cutoffs.

The director has a strict adaptation budget so it cannot counter every player success.

3. Witness, evidence, and rumor
Perception is not knowledge
A witness pipeline has five stages:

SENSATION → ATTENTION → INTERPRETATION → CLAIM → PROPAGATION
A camera can sense and record but not interpret motive. A frightened civilian may attend poorly. A trained guard may identify equipment. A faction analyst can combine partial claims. Each stage introduces uncertainty.

Claim model
A claim states:

subject, predicate, value, confidence, valence,
source, source-chain, origin-event, location, identity-key,
created-tick, updated-tick, tags
Examples:

“The Night Courier was seen near the archive,” confidence 0.72.

“Someone using impossible speed attacked Ward officers,” confidence 0.55.

“The player protects Saint Orra residents,” confidence 0.41, positive valence.

“The red coupe is linked to recent thefts,” confidence 0.83.

Claims can conflict. NPCs and factions hold beliefs, not omniscient truth.

Witness generation
Witness probability depends on:

event visibility,

distance and occlusion,

lighting and weather,

current attention,

sensory capability,

fear and stress,

disguise quality,

event novelty,

and whether the witness has a reason to look away.

Do not simulate every ray for offscreen actors. Near the player, use game perception. Outside the fidelity bubble, use aggregate probabilities.

Propagation
Rumors move along social edges:

family, team, faction, neighborhood, media, institutional records, black market.

Trust in source changes confidence.

Retelling decays detail and can amplify valence.

Some traits distort toward threat, scandal, loyalty, or self-interest.

Claims can merge when subject, predicate, location, and time are compatible.

False claims remain playable; truth is not automatically privileged.

Cleanup
The player can:

remove or alter physical evidence,

persuade, intimidate, bribe, rescue, discredit, or recruit witnesses,

destroy records,

seed an alternate identity,

frame another faction,

retire a vehicle or disguise,

call a favor,

accept public notoriety,

or convert fear into deliberate reputation.

Cleanup is gameplay, not an after-action tax. Keep it optional by allowing players to ride the consequences.

4. Faction belief and agenda
Each faction has:

power, cohesion, resources, territory, awareness,
trust(player), fear(player), respect(player),
agenda, red-lines, needs, active operations, liabilities
Trust, fear, and respect are separate. High fear can open coercive access while making betrayal and coalition formation more likely. High trust can enable aid but also create expectations. Respect comes from demonstrated competence aligned with faction values.

Agenda tick
At a low simulation frequency:

Evaluate needs and threats.

Generate candidate operations.

Reserve resources.

Select operations using utility scores and strategic personality.

Execute abstractly or instantiate locally when near the player.

Emit events.

Update district assets and beliefs.

Surface at least one readable signal to the player.

No invisible world-state change should matter greatly without a clue: changed guards, prices, graffiti, rumors, news, patrol routes, ambient dialogue, a message, or a physical transformation.

5. Opportunity director
Procedural generation should arrange authored meaning, not synthesize sludge.

Opportunity template anatomy
premise
required world predicates
role slots
locations and spatial affordances
objective graph
complications
revelations
phase transitions
resolution methods
fail-forward outcomes
consequence outputs
reward forms
cooldown and novelty tags
authored lines / bark sets
Scoring
Candidate score combines:

0.22 pressure relevance
0.18 faction agenda relevance
0.16 novelty / cooldown
0.12 player-style support
0.08 gentle counterpoint
0.08 geographic coherence
0.08 relationship relevance
0.05 resource-state fit
0.03 authored priority
Weights are starting values only. The balance harness measures whether they create dominant loops.

“Style support” gives the player chances to express mastery. “Counterpoint” introduces occasional friction. Counterpoint is capped below support and cannot select a hard counter twice consecutively.

Objective graph
Represent objectives as a directed graph with typed edges:

requires

reveals

invalidates

escalates

substitutes

settles

creates_debt

moves_target

changes_owner

At least one noncombat resolution and one fail-forward resolution should exist for reusable templates unless fiction forbids it.

Example: The Ledger Breathes
Premise: a mobile ledger is being transferred between factions.

Possible entries:

intercept the courier vehicle,

infiltrate the transfer location,

persuade the courier to duplicate it,

replace the ledger with a marked copy,

buy the route from a dispatcher,

trigger a rival attack and steal it during confusion.

Complications:

the ledger is a person with eidetic memory,

a player ally is named in it,

it is bait,

the courier wants extraction,

the “ledger” contains false records planted for a faction purge.

Resolutions:

deliver, duplicate, destroy, publish, alter, ransom, or protect.

Fail-forward:

the target escapes but a route and identity are learned,

the player obtains a partial record,

a faction believes the player has the full ledger,

an ally takes possession and incurs debt.

The template becomes replayable because world state fills the semantic slots.

6. Player-style inference
Track method contribution at event resolution, not raw input count. Use an exponentially decayed vector.

Contribution examples
Damage alone is not always FORCE. A precisely timed environmental collapse prepared through systems can split contribution.

Remaining unseen while bypassing a lock contributes STEALTH and SYSTEMS.

A chase escape through route mastery contributes MOBILITY even without a vehicle.

Delegating a hit through a favor contributes INFLUENCE and perhaps FORCE at lower strength.

Talking an enemy down after visibly demonstrating power can contribute INFLUENCE plus FORCE.

Entropy
Normalize the style vector and compute Shannon entropy. Low entropy indicates specialization; high entropy indicates hybrid play. Use this only for recommendations and content variety.

Adaptation budget
Per three opportunities:

at least one strongly supports the dominant style,

at most one contains a soft counter,

at least one creates a bridge to another method,

no hard counter may erase the player’s signature verb,

and all counter information must be discoverable before commitment.

The world should learn the player without becoming a spiteful dungeon master.

7. Combat
Contact model
Use three interacting layers:

Health/condition: long-term survivability.

Poise/posture: short-term ability to resist interruption.

Guard/armor: directional or equipment-specific mitigation.

A hit resolves:

attack definition
→ active hit volume
→ contact query
→ material/body-zone response
→ damage + poise + impulse
→ reaction selection
→ audiovisual packet
→ event emission
Reaction selection
Select animation by:

hit direction in victim space,

magnitude band,

body zone,

current locomotion,

stance,

remaining poise,

surface/obstacle context,

weapon family,

and whether the hit is lethal/nonlethal.

Use additive reactions for light hits, montage/partial-body for medium, full-body for guard break, and controlled ragdoll blend for exceptional impulse. Prevent reaction loops with diminishing interrupt authority.

Combat readability
Enemy roles have silhouette, movement, bark, and effect-language differences.

Telegraphs survive motion blur and dark scenes.

Attack commitment is readable without becoming slow.

The player can identify why damage occurred.

Offscreen threats use audio and camera-space cues.

Hit pause is short, selective, and disabled or reduced for rapid multi-hit attacks.

Camera shake scales by distance and can be independently reduced.

VFX do not hide the next attack.

Enemy roles
Use compositional squads:

Anchor: holds space, protects.

Flanker: changes angle, pressures cover.

Hunter: tracks hidden or fleeing players.

Controller: denies routes, deploys hazards.

Negotiator: creates surrender, hostage, or social exits.

Specialist: responds to anomaly or signature powers.

Civilian/Dependent: changes acceptable collateral and witness state.

Enemies should coordinate through intent reservations so five actors do not choose the same flank or attack slot.

Injury
Injury should create short arcs, not permanent annoyance:

arm injury: recoil/interaction cost; enables alternate grip or treatment.

leg injury: acceleration/landing penalty; enables brace, vehicle, ally support.

concussion: information noise; avoid literal screen blur as default.

bleeding: time pressure; multiple treatment qualities.

supernatural/technical instability: power side effects.

Allow accessibility settings to convert handling penalties into resource penalties.

8. Stealth
Stealth uses suspicion states rather than binary detection:

UNCONCERNED → CURIOUS → SUSPICIOUS → SEARCHING → CONFIRMED → COORDINATED
Actors retain beliefs about last known position, identity, method, and likely goal. Search behavior covers exits, objectives, allies, hiding affordances, and evidence—not only the last visible point.

The player can manipulate:

light,

sound,

access permissions,

identity,

evidence,

schedule,

faction expectations,

and the story witnesses tell themselves.

Bodies are not the only evidence. Missing objects, disabled cameras, open doors, broken routines, unusual vehicles, social anomalies, and contradictory records matter at tuned levels.

9. Social play
Conversation is an encounter system, not a colored key check.

Social state
Each participant has:

goal, fear, need, red-line, current stance,
trust, fear, respect, leverage-known, audience, time pressure

Presentation doctrine
The game should look expensive because materials, light, motion, sound, and reactions agree—not because every effect is maximal.

The visual target is legible nocturnal density:

deep values without crushed information,

selective saturated light,

material-specific wet response,

stable temporal image,

disciplined emissive intensity,

faces and hands readable at conversation distance,

silhouettes readable at combat distance,

Rendering hierarchy
Spend in this order:

camera and temporal stability,

lighting composition,

material response,

character shading,

contact grounding,

animation and reaction,

decals and localized VFX,

atmospheric depth,

expensive secondary effects.

A cheaper effect with correct timing and placement beats a global expensive effect.

Every major surface family declares:

roughness range

macro variation
edge/wear behavior
wet response
decal response
damage response
footstep/impact audio class

05 — BALANCE, TELEMETRY, AND QA
Telemetry philosophy
Telemetry answers design questions; it does not replace observation. Collect the minimum needed, aggregate where possible, make it optional, and document it plainly.

Local development telemetry is mandatory for debugging. Shipped analytics require privacy review and player control.

Event-derived metrics
Feel
input-to-action latency proxy,

buffered-input success,

camera collision duration,

unexpected grounded-state transitions,

failed interaction attempts,

hit confirmation timing,

control-lock duration,

and recovery interruption rate.

Encounter
method contribution,

damage/poise exchange,

detection path,

time in suspicion states,

route changes,

resource spend,

injury,

witness/evidence residue,

retry/continue choice,

and outcome.

Campaign
style vector and entropy,

faction trust/fear/respect,

debt creation/settlement,

district pressure,

opportunity acceptance and abandonment,

infrastructure use,

identity compromise,

and recovery time after setbacks.

Fun proxies
No scalar measures fun. Use a panel:

agency: number of meaningful viable actions at decision points,

clarity: percentage of setbacks players can correctly explain,

expression: divergence of method profiles across players,

mastery: improvement without raw stat increase,

surprise: novel but retrospectively coherent outcomes,

recovery: fraction of failures continued rather than reloaded,

tempo: time between meaningful state changes,

residue yield: future opportunities generated per resolved job.

These proxies identify questions for playtests.

Dominant-strategy detection
A strategy is suspect when it combines:

high success,

high reward,

low pressure,

low resource cost,

low execution demand,

and broad applicability.

The included simulator compares method profiles over seeds. It does not prove human balance; it finds structural imbalance before content production multiplies it.

Dominance score
Normalize each metric and estimate:

dominance =
  + success_z
  + reward_z
  - exposure_z
  - injury_z
  - debt_z
  - resource_cost_z
  - decision_entropy_penalty
Investigate, do not auto-nerf. A method may look dominant because opportunity templates overrepresent its affordances.

Adaptation fairness
Track:

support opportunities per dominant style,

soft counters,

hard counters,

counter warning time,

alternate responses,

repeated counter streak,

and success after adaptation.

Reject tuning where the director’s knowledge exceeds in-world faction knowledge.

Determinism
Record:

root seed,

content version,

tuning hash,

event-log checkpoint,

random stream names,

and platform/build.

Random streams are partitioned by domain:

world_init
faction_agenda
opportunity_selection
opportunity_slots
rumor_distortion
npc_decision:<npc-id>
loot_or_reward
cosmetic_only
Cosmetic randomness may diverge. Gameplay randomness may not consume from cosmetic streams.

Test matrix
Character movement
flat, slope, step, ledge, moving platform,

high/low frame rate,

variable render rate with fixed physics,

collision corner,

low ceiling,

crouch transition,

knockback,

root-motion action,

save/load mid-state where supported.

Combat
each damage class/material pair,

simultaneous hits,

multi-hit,

poise break,

armor break,

nonlethal,

ragdoll near stairs/walls/vehicles,

offscreen threat,

friendly/civilian context,

and input during reaction.

AI
sensory loss,

conflicting claims,

partial identity,

duplicate event,

faction ally/enemy,

unreachable target,

reserved action conflict,

search timeout,

surrender,

and simulation LOD transition.

Mission/opportunity
every entry,

every objective substitution,

target death/escape,

item lost/destroyed,

faction ownership change,

player leaves district,

save/load each phase,

content hot reload in editor if supported,

and all fail-forward exits.

Save
clean save,

repeated round trip,

interrupted write,

old-version migration,

missing optional content,

invalid content ID,

active pursuit,

active conversation,

active opportunity,

and large claim graph.

Fuzzing
Use property tests or randomized scripts for:

event order permutations where order should not matter,

duplicate events,

missing optional metadata,

extreme pressure values,

invalid relationships,

empty candidate sets,

cyclic objective graphs,

and save migration boundaries.

Invariants:

values remain finite,

bounded fields remain bounded,

no negative inventory/resources unless explicitly debt,

no opportunity requires unavailable roles,

no actor is both alive and resolved dead,

no identity heat leaks to unrelated identities without a claim,

and no event applies twice.

Playtest protocol
Each vertical-slice test includes:

interview asking the player to explain state,

alternate-style second run,

continuation after a forced setback,

five-minute free experimentation,

and a map/network reconstruction from memory.

Observe before explaining. The reconstruction reveals whether the city is legible.

Quality dashboard

and content validation failures.

06 — CONTENT BIBLE: THE CITY AS AN ARGUMENT
This is an original modular setting scaffold. Rename, replace, or discard it when binding to the repository. Its purpose is to demonstrate how factions, districts, opportunities, art, and mechanics can share one grammar.

Tone
Intimate urban dread crossed with kinetic competence. The city is not merely corrupt; it is overinterpreted. Cameras, gossip, bureaucracy, desire, and myth all compete to decide what happened.

Humor is dry, local, and human. Horror arrives through implication and consequence more often than gore. Spectacle is rare enough to remain spectacular.

The city
Dockglass
A logistics waterfront of mirrored warehouses, flooded service tunnels, elevated roads, labor bars, and private customs offices. It supports vehicles, vertical infiltration, industrial hazards, union politics, and evidence disposal.

Material language: wet concrete, corrugated metal, sodium amber, cyan security light, oil rainbow, red gantry beacons.

Saint Orra
A dense residential and devotional quarter built around clinics, markets, courtyards, old infrastructure, and contested community institutions. It supports social play, protection, favors, hidden routes, and consequences with faces.

Material language: worn stone, tile, fabric awnings, warm windows, green pharmacy light, hand-painted signs, rain-dark wood.

The Meridian
Corporate, civic, and nightlife towers connected by transit halls, parking structures, hotels, service floors, and controlled plazas. It supports multiple identities, surveillance, formal status, elevators, rooftops, and public narrative.

Material language: black glass, pale stone, brushed alloy, magenta event light, disciplined greenery, clean reflective water.

The Underline
Not a full district at first. It is the connective anatomy beneath all three: maintenance routes, transit service zones, data trunks, abandoned stations, drainage, and forgotten rooms. It becomes an endgame/composite space shaped by prior district state.

Factions
Civic Ward
Public safety institution with genuine protectors, political operators, exhausted responders, and specialist containment units.

Values: order, legibility, continuity.
Need: legitimacy and controllable narratives.
Red lines: public humiliation, unexplained mass casualty.
Mechanical signature: records, checkpoints, coordinated response, evidence chains.

Glass Serpents
A distributed logistics and theft network that treats routes as property.

Values: competence, reciprocity, deniable profit.
Need: reliable movement and neutral brokers.
Red lines: theft without acknowledgment, disruption of trusted routes.
Mechanical signature: vehicles, smuggling, pursuit, hidden cargo, route intelligence.

Choir of Static
A loose culture of signal artists, occult technicians, pirate broadcasters, and anomaly enthusiasts.

Values: revelation, pattern, transgression.
Need: access to forbidden events and audiences.
Red lines: suppression, boring lies.
Mechanical signature: surveillance, rumor amplification, strange abilities, false narratives.

Saint Orra Mutual
Clinics, tenants, kitchens, couriers, advocates, and protectors held together by obligation rather than command.

Values: care, memory, local autonomy.
Need: supplies, safety, reliable people.
Red lines: predation on dependents, abandonment after promises.
Mechanical signature: safe routes, treatment, witnesses, community concealment, accumulating debt.

Meridian Compact
Corporate and civic houses that trade access, liability, and reputation.

Values: continuity, status, enforceable agreement.
Need: plausible stability and information asymmetry.
Red lines: uncontrolled disclosure, contracts made meaningless.
Mechanical signature: identities, credentials, legal cover, high-value social encounters.

Pale Office
A small specialist organization that investigates events which do not fit institutional categories.

Values: containment, proof, patience.
Need: clean evidence and predictable anomalies.
Red lines: contamination of evidence, public spectacle.
Mechanical signature: adaptive hunters, forensic linkage, anti-power tools, quiet offers.

Relationship construction
Every named relationship character needs:

a practical role,

a private need,

a contradiction,

a capability the player cannot simply buy,

a red line,

a memory trigger,

one scene of competence,

one scene of vulnerability,

and at least two possible future roles.

Do not write “quest dispensers.” A contact should be able to become ally, rival, dependent, target, witness, successor, or absence.

Environmental storytelling grammar
Each reusable space carries three layers:

Function: what people do here.

Ownership: who controls, maintains, watches, or claims it.

Recent event: what changed.

A clinic under shortage differs physically from the same clinic under Ward protection or Serpent smuggling. Use modular state kits: signs, guards, stock levels, barricades, lighting, ambient lines, stains, vehicles, and access changes.

Bark grammar
Barks communicate state:

[faction voice] + [observed fact] + [belief/uncertainty] + [intent]
Examples are assembled from authored fragments with strict tone tags. Avoid repeating exact player summaries. Uncertainty should be audible:

“Red coat, maybe. Moved like the cameras skipped.”

“Ward says it was gang work. Ward says many things.”

“That route is burned. Use the kitchen stairs and owe me.”

Barks may create claims when heard by relevant actors.

Opportunity families
Transfer
Move person, object, data, or obligation across contested space.

Extraction
Remove a willing, unwilling, injured, compromised, or ambiguous target.

Suppression
Stop evidence, a broadcast, an operation, a rumor, a ritual, or a witness chain.

Substitution
Replace an object, identity, record, leader, route, or public explanation.

Arbitration
Resolve a conflict through proof, status, coercion, sacrifice, or deal structure.

Containment
Keep a dangerous state within bounds while factions compete over it.

Demonstration
Make a public or selective example to change belief.

Each family supports multiple methods and residue profiles.

Signature mission construction
A signature mission gets one handcrafted impossibility: a spatial, social, physical, or conceptual situation that templates cannot reproduce. It still binds to systemic consequences.

Examples:

a negotiation conducted while a high-rise slowly evacuates,

a pursuit through three ownership layers of the same transit line,

a clinic defense where “winning” by violence destroys its legitimacy,

an archive whose records alter security behavior as they are rewritten,

or a faction summit where each identity the player used is separately invited.

The authored impossible moment is the jewel; the systemic aftermath is its setting.

Dialogue doctrine
Characters speak from goals, not lore delivery.

Exposition is traded, contested, misremembered, or demonstrated.

The player can leave many conversations.

Silence, timing, audience, and prior residue matter.

A successful social action changes the situation.

A failed social action reveals information or creates a cost.

Repeated questions receive human responses.

Important choices do not hide behind arbitrary timeout unless time pressure is fictional and readable.

End states
Do not reduce the ending to faction color.

Synthesize epilogue dimensions:

who controls movement,

who controls the public story,

which district was protected or exploited,

what the player became known for,

which debts were honored,

who learned the truth,

how extraordinary power entered ordinary institutions,

and what legacy remains for the next protagonist.

The ending should feel like a proof assembled from play.
""")

write("reference/nightglass_sim/rng.py", r'''
from future import annotations

from dataclasses import dataclass
from typing import Iterable, Sequence, TypeVar

MASK64 = (1 << 64) - 1
T = TypeVar("T")

def stable_hash64(value: object) -> int:
"""Stable FNV-1a hash. Never use Python's randomized hash for replay state."""
data = str(value).encode("utf-8")
h = 0xCBF29CE484222325
for byte in data:
h ^= byte
h = (h * 0x100000001B3) & MASK64
return h

@dataclass
class SplitMix64:
"""Small deterministic generator suitable for simulation stream partitioning."""
state: int

def __post_init__(self) -> None:
    self.state &= MASK64

def next_u64(self) -> int:
    self.state = (self.state + 0x9E3779B97F4A7C15) & MASK64
    z = self.state
    z = ((z ^ (z >> 30)) * 0xBF58476D1CE4E5B9) & MASK64
    z = ((z ^ (z >> 27)) * 0x94D049BB133111EB) & MASK64
    return (z ^ (z >> 31)) & MASK64

def random(self) -> float:
    return (self.next_u64() >> 11) * (1.0 / (1 << 53))

def uniform(self, lo: float, hi: float) -> float:
    return lo + (hi - lo) * self.random()

def randint(self, lo: int, hi: int) -> int:
    if hi < lo:
        raise ValueError("hi must be >= lo")
    span = hi - lo + 1
    # Rejection avoids modulo bias.
    limit = MASK64 - (MASK64 % span)
    while True:
        value = self.next_u64()
        if value < limit:
            return lo + (value % span)

def choice(self, values: Sequence[T]) -> T:
    if not values:
        raise ValueError("cannot choose from an empty sequence")
    return values[self.randint(0, len(values) - 1)]

def weighted_choice(self, values: Sequence[T], weights: Sequence[float]) -> T:
    if len(values) != len(weights) or not values:
        raise ValueError("values and weights must have equal nonzero length")
    total = sum(max(0.0, w) for w in weights)
    if total <= 0.0:
        return self.choice(values)
    point = self.uniform(0.0, total)
    cumulative = 0.0
    for value, weight in zip(values, weights):
        cumulative += max(0.0, weight)
        if point <= cumulative:
            return value
    return values[-1]

def fork(self, *parts: object) -> "SplitMix64":
    seed = self.state
    for part in parts:
        seed ^= stable_hash64(part)
        # One avalanche step without consuming the parent stream.
        seed = (seed + 0x9E3779B97F4A7C15) & MASK64
        z = seed
        z = ((z ^ (z >> 30)) * 0xBF58476D1CE4E5B9) & MASK64
        z = ((z ^ (z >> 27)) * 0x94D049BB133111EB) & MASK64
        seed = (z ^ (z >> 31)) & MASK64
    return SplitMix64(seed)
''')

write("reference/nightglass_sim/models.py", r'''
from future import annotations

from dataclasses import dataclass, field
from typing import Any, Dict, List, Mapping, MutableMapping, Optional, Sequence, Set

METHODS = ("force", "stealth", "influence", "mobility", "systems")
PRESSURE_CHANNELS = ("exposure", "heat", "need", "injury", "debt", "anomaly", "volatility")

def clamp01(value: float) -> float:
return max(0.0, min(1.0, float(value)))

@dataclass
class PressureState:
exposure: float = 0.0
heat: float = 0.0
need: float = 0.0
injury: float = 0.0
debt: float = 0.0
anomaly: float = 0.0
volatility: float = 0.0
local_heat: Dict[str, float] = field(default_factory=dict)
identity_exposure: Dict[str, float] = field(default_factory=dict)

def add(self, channel: str, amount: float, *, district_id: str = "", identity_key: str = "") -> None:
    if channel not in PRESSURE_CHANNELS:
        raise KeyError(f"unknown pressure channel: {channel}")
    setattr(self, channel, clamp01(getattr(self, channel) + amount))
    if channel == "heat" and district_id:
        self.local_heat[district_id] = clamp01(self.local_heat.get(district_id, 0.0) + amount)
    if channel == "exposure" and identity_key:
        self.identity_exposure[identity_key] = clamp01(
            self.identity_exposure.get(identity_key, 0.0) + amount
        )

def decay(self, rates: Mapping[str, float]) -> None:
    for channel in PRESSURE_CHANNELS:
        rate = max(0.0, float(rates.get(channel, 0.0)))
        setattr(self, channel, clamp01(getattr(self, channel) - rate))
    for key in list(self.local_heat):
        self.local_heat[key] = clamp01(self.local_heat[key] - rates.get("heat", 0.0))
        if self.local_heat[key] <= 1e-6:
            del self.local_heat[key]

from future import annotations

import math
from dataclasses import dataclass, field
from typing import Dict, Mapping

from .models import METHODS, WorldEvent, clamp01

@dataclass
class PlaystyleTracker:
decay: float = 0.94
meaningful_floor: float = 0.08
values: Dict[str, float] = field(default_factory=lambda: {m: 0.0 for m in METHODS})
meaningful_events: int = 0

def record(self, event: WorldEvent, split: Mapping[str, float] | None = None) -> None:
    for method in METHODS:
        self.values[method] *= self.decay

    contributions = dict(split or {})
    if not contributions:
        if event.method in METHODS:
            contributions[event.method] = 1.0

    total = sum(max(0.0, v) for v in contributions.values())
    magnitude = clamp01(event.intensity) * (0.5 + 0.5 * clamp01(event.visibility))
    if total <= 0.0 or magnitude < self.meaningful_floor:
        return

    for method, value in contributions.items():
        if method in self.values:
            self.values[method] += magnitude * max(0.0, value) / total
    self.meaningful_events += 1

def normalized(self) -> Dict[str, float]:
    total = sum(max(0.0, v) for v in self.values.values())
    if total <= 1e-12:
        return {m: 1.0 / len(METHODS) for m in METHODS}
    return {m: max(0.0, self.values[m]) / total for m in METHODS}

def entropy(self) -> float:
    distribution = self.normalized()
    h = -sum(p * math.log(p, 2) for p in distribution.values() if p > 0.0)
    return h / math.log(len(METHODS), 2)

def dominant(self) -> tuple[str, float]:
    distribution = self.normalized()
    method = max(distribution, key=distribution.get)
    return method, distribution[method]

def support_fit(self, methods: set[str]) -> float:
    distribution = self.normalized()
    return clamp01(sum(distribution.get(m, 0.0) for m in methods))

def counterpoint(self, methods: set[str]) -> float:
    dominant_method, dominance = self.dominant()
    if dominant_method in methods:
        return 0.0
    # Counterpoint grows only when the player is specialized, and is deliberately capped.
    return min(0.45, max(0.0, dominance - 0.35))
''')

from future import annotations

from dataclasses import dataclass, field
from typing import Callable, Dict, Generic, Iterable, List, Mapping, Sequence, TypeVar

from .models import clamp01
from .rng import SplitMix64

ContextT = TypeVar("ContextT")

def linear(x: float) -> float:
return clamp01(x)

def inverse(x: float) -> float:
return 1.0 - clamp01(x)

def smoothstep(x: float) -> float:
x = clamp01(x)
return x * x * (3.0 - 2.0 * x)

def threshold(x: float, edge: float = 0.5, softness: float = 0.1) -> float:
if softness <= 0:
return 1.0 if x >= edge else 0.0
lo = edge - softness
hi = edge + softness
return smoothstep((x - lo) / max(1e-9, hi - lo))

@dataclass
class Consideration(Generic[ContextT]):
name: str
input_fn: Callable[[ContextT], float]
curve: Callable[[float], float] = linear
weight: float = 1.0

@dataclass
class UtilityAction(Generic[ContextT]):
id: str
base_score: float
considerations: List[Consideration[ContextT]]
commitment_bonus: float = 0.0
personality_key: str = ""
tags: set[str] = field(default_factory=set)

def score(
    self,
    context: ContextT,
    *,
    personality: Mapping[str, float] | None = None,
    committed_action_id: str = "",
    rng: SplitMix64 | None = None,
    noise: float = 0.0,
) -> tuple[float, Dict[str, float]]:
    personality = personality or {}
    factors: Dict[str, float] = {}
    weighted_log_sum = 0.0
    total_weight = 0.0

    for consideration in self.considerations:
        raw = clamp01(consideration.input_fn(context))
        curved = max(1e-5, clamp01(consideration.curve(raw)))
        weight = max(0.0, consideration.weight)
        factors[consideration.name] = curved
        weighted_log_sum += weight * __import__("math").log(curved)
        total_weight += weight

    composite = __import__("math").exp(weighted_log_sum / max(1e-9, total_weight))
    score = max(0.0, self.base_score) * composite

    if self.personality_key:
        score *= 0.65 + 0.7 * clamp01(personality.get(self.personality_key, 0.5))
    if committed_action_id == self.id:
        score += max(0.0, self.commitment_bonus)
    if rng is not None and noise > 0.0:
        score *= 1.0 + rng.uniform(-noise, noise)

    factors["composite"] = composite
    factors["final"] = max(0.0, score)
    return max(0.0, score), factors
''')

from future import annotations

from dataclasses import dataclass
from typing import Dict, Iterable, List, Mapping, Tuple

from .models import Claim, NpcState, WorldEvent, WorldState, clamp01
from .rng import SplitMix64, stable_hash64

@dataclass
class RumorConfig:
base_witness_threshold: float = 0.25
confidence_decay_per_tick: float = 0.012
propagation_decay: float = 0.18
merge_threshold: float = 0.62
minimum_retained_confidence: float = 0.08
max_claims_per_npc: int = 64

class RumorEngine:
def init(self, config: RumorConfig | None = None) -> None:
self.config = config or RumorConfig()

def witness_event(self, world: WorldState, event: WorldEvent) -> List[Claim]:
    claims: List[Claim] = []
    for npc in sorted(world.npcs.values(), key=lambda n: n.id):
        if not npc.alive or not npc.active or npc.district_id != event.district_id:
            continue
        if npc.id == event.actor_id or npc.id in event.target_ids:
            continue

        attentiveness = clamp01(npc.traits.get("attentiveness", 0.5))
        stress = clamp01(npc.traits.get("stress", 0.0))
        fear = clamp01(npc.traits.get("fear", 0.0))
        look_away = clamp01(npc.traits.get("look_away", 0.0))
        perception = event.visibility * (0.35 + 0.65 * attentiveness)
        perception *= (1.0 - 0.35 * stress) * (1.0 - 0.25 * fear) * (1.0 - 0.7 * look_away)

        rng = SplitMix64(world.root_seed).fork("witness", event.event_id, npc.id)
        threshold = self.config.base_witness_threshold + rng.uniform(-0.08, 0.08)
        if perception < threshold:
            continue

        identity_known = event.identity_key not in ("", "unknown")
        disguise_ambiguity = clamp01(float(event.metadata.get("identity_ambiguity", 0.0)))
        identity_confidence = perception * (1.0 - 0.7 * disguise_ambiguity)
        identity_key = event.identity_key if identity_known and identity_confidence >= 0.28 else "unknown"

        confidence = clamp01(0.25 + 0.65 * perception + rng.uniform(-0.08, 0.08))
        valence = self._valence_for(event, npc)
        claim = Claim(
            claim_id=f"claim_{stable_hash64((event.event_id, npc.id, 'method')):016x}",
            subject_id=event.actor_id,
            predicate="used_method",
            value=event.method,
            confidence=confidence,
            valence=valence,
            source_id=npc.id,
            source_chain=[npc.id],
            origin_event_id=event.event_id,
            district_id=event.district_id,
            identity_key=identity_key,
            created_tick=world.tick,
            last_updated_tick=world.tick,
            tags=set(event.tags) | {"eyewitness"},
        )
        self._store_claim(npc, claim)
        claims.append(claim)

        if identity_key != "unknown":
            identity_claim = Claim(
                claim_id=f"claim_{stable_hash64((event.event_id, npc.id, 'identity')):016x}",
                subject_id=event.actor_id,
                predicate="identity_link",
                value=identity_key,
                confidence=clamp01(confidence * identity_confidence),
                valence=valence,
                source_id=npc.id,
                source_chain=[npc.id],
                origin_event_id=event.event_id,
                district_id=event.district_id,
                identity_key=identity_key,
                created_tick=world.tick,
                last_updated_tick=world.tick,
                tags={"eyewitness", "identity"},
            )
            self._store_claim(npc, identity_claim)
            claims.append(identity_claim)
    return claims

def propagate(self, world: WorldState) -> List[tuple[str, Claim]]:
    transfers: List[tuple[str, Claim]] = []
    npcs = sorted((n for n in world.npcs.values() if n.alive), key=lambda n: n.id)

    for source in npcs:
        if not source.knowledge:
            continue
        for target_id, relationship in sorted(source.relationships.items()):
            target = world.npcs.get(target_id)
            if target is None or not target.alive:
                continue
            if source.district_id != target.district_id and source.faction_id != target.faction_id:
                continue

            trust = clamp01((relationship + 1.0) * 0.5)
            gossip = clamp01(source.traits.get("gossip", 0.5))
            restraint = clamp01(source.traits.get("restraint", 0.0))
            rng = SplitMix64(world.root_seed).fork("propagate", world.tick, source.id, target.id)
            if rng.random() > 0.08 + 0.42 * gossip * (1.0 - restraint):
                continue

            source_claim = max(source.knowledge.values(), key=lambda c: c.confidence)
            distortion = rng.uniform(-0.08, 0.08) * (1.0 - trust)
            threat_bias = clamp01(target.traits.get("threat_bias", 0.5))
            valence = clamp01((source_claim.valence + 1.0) * 0.5)
            valence = 2.0 * clamp01(valence + (threat_bias - 0.5) * 0.08) - 1.0
            confidence = clamp01(
                source_claim.confidence
                * (1.0 - self.config.propagation_decay)
                * (0.45 + 0.55 * trust)
                + distortion
            )
            if confidence < self.config.minimum_retained_confidence:
                continue

            propagated = Claim(
                claim_id=f"claim_{stable_hash64((source_claim.claim_id, target.id, world.tick)):016x}",
                subject_id=source_claim.subject_id,
                predicate=source_claim.predicate,
                value=source_claim.value,
                confidence=confidence,
                valence=valence,
                source_id=source.id,
                source_chain=(source_claim.source_chain + [source.id])[-8:],
                origin_event_id=source_claim.origin_event_id,
                district_id=source_claim.district_id,
                identity_key=source_claim.identity_key,
                created_tick=world.tick,
                last_updated_tick=world.tick,
                tags=set(source_claim.tags) | {"hearsay"},
            )
            self._store_claim(target, propagated)
            transfers.append((target.id, propagated))
    return transfers

def decay(self, world: WorldState) -> None:
    for npc in world.npcs.values():
        remove: List[str] = []
        for claim_id, claim in npc.knowledge.items():
            age = max(0, world.tick - claim.last_updated_tick)
            # One decay quantum per invocation; caller controls cadence.
            claim.confidence = clamp01(claim.confidence - self.config.confidence_decay_per_tick)
            claim.last_updated_tick = world.tick
            if claim.confidence < self.config.minimum_retained_confidence:
                remove.append(claim_id)
        for claim_id in remove:
            del npc.knowledge[claim_id]

def faction_beliefs(self, world: WorldState, faction_id: str) -> Dict[str, float]:
    belief = {"trust": 0.0, "fear": 0.0, "awareness": 0.0}
    members = [n for n in world.npcs.values() if n.faction_id == faction_id and n.alive]
    if not members:
        return belief
    weight_sum = 0.0
    for npc in members:
        for claim in npc.knowledge.values():
            weight = claim.confidence * (1.0 if "eyewitness" in claim.tags else 0.65)
            belief["awareness"] += weight
            if claim.valence < 0:
                belief["fear"] += weight * -claim.valence
            else:
                belief["trust"] += weight * claim.valence
            weight_sum += weight
    if weight_sum > 0:
        belief["trust"] = clamp01(belief["trust"] / weight_sum)
        belief["fear"] = clamp01(belief["fear"] / weight_sum)
        belief["awareness"] = clamp01(weight_sum / max(1.0, len(members)))
    return belief

def _store_claim(self, npc: NpcState, incoming: Claim) -> None:
    compatible = [
        existing for existing in npc.knowledge.values()
        if existing.key() == incoming.key()
    ]
    if compatible:
        existing = max(compatible, key=lambda c: c.confidence)
        agreement = 1.0 - abs(existing.valence - incoming.valence) * 0.25
        combined = 1.0 - (1.0 - existing.confidence) * (1.0 - incoming.confidence * agreement)
        existing.confidence = clamp01(combined)
        existing.valence = max(-1.0, min(1.0, (existing.valence + incoming.valence) * 0.5))
        existing.last_updated_tick = incoming.last_updated_tick
        existing.source_chain = (existing.source_chain + incoming.source_chain)[-8:]
        existing.tags |= incoming.tags
        return

    npc.knowledge[incoming.claim_id] = incoming
    if len(npc.knowledge) > self.config.max_claims_per_npc:
        weakest = min(npc.knowledge.values(), key=lambda c: (c.confidence, c.created_tick))
        del npc.knowledge[weakest.claim_id]

@staticmethod
def _valence_for(event: WorldEvent, npc: NpcState) -> float:
    protective = "protective" in event.tags or "rescue" in event.tags
    lethal = "lethal" in event.tags
    collateral = "collateral" in event.tags
    base = 0.35 if protective else -0.25
    if lethal:
        base -= 0.35
    if collateral:
        base -= 0.4
    loyalty = npc.traits.get("loyalty_to_actor", 0.0)
    threat_bias = npc.traits.get("threat_bias", 0.5)
    return max(-1.0, min(1.0, base + 0.35 * loyalty - 0.2 * threat_bias))
''')

from future import annotations

import math
from dataclasses import dataclass
from typing import Dict, Iterable, List, Mapping, Sequence, Tuple

from .models import (
METHODS,
Opportunity,
OpportunityTemplate,
WorldState,
clamp01,
)
from .playstyle import PlaystyleTracker
from .rng import SplitMix64, stable_hash64

@dataclass
class DirectorConfig:
weights: Dict[str, float]
temperature: float = 0.18

@staticmethod
def default() -> "DirectorConfig":
    return DirectorConfig(weights={
        "pressure_relevance": 0.22,
        "faction_agenda": 0.18,
        "novelty": 0.16,
        "style_support": 0.12,
        "counterpoint": 0.08,
        "geography": 0.08,
        "relationship": 0.08,
        "resource_fit": 0.05,
        "authored_priority": 0.03,
    })
class OpportunityDirector:
def init(self, templates: Sequence[OpportunityTemplate], config: DirectorConfig | None = None) -> None:
self.templates = list(templates)
self.config = config or DirectorConfig.default()

def candidates(
    self,
    world: WorldState,
    style: PlaystyleTracker,
) -> List[tuple[OpportunityTemplate, str, str, float, Dict[str, float]]]:
    candidates: List[tuple[OpportunityTemplate, str, str, float, Dict[str, float]]] = []
    for template in self.templates:
        last_tick = world.recent_template_ticks.get(template.id, -10**9)
        cooldown_ratio = clamp01((world.tick - last_tick) / max(1, template.cooldown_ticks))
        if cooldown_ratio <= 0.0:
            continue

        for district in world.districts.values():
            if template.required_tags and not template.required_tags.issubset(district.tags):
                continue
            if template.forbidden_tags & district.tags:
                continue

            faction_id, faction_fit = self._best_faction(world, district.id)
            if not faction_id:
                continue
            faction = world.factions[faction_id]

            pressure_relevance = self._pressure_relevance(world, template)
            novelty = 0.35 + 0.65 * cooldown_ratio
            style_support = style.support_fit(template.resolution_methods)
            counterpoint = style.counterpoint(template.resolution_methods)
            geography = 1.0 if district.id == world.player.district_id else (
                0.72 if district.id in world.districts[world.player.district_id].adjacent else 0.35
            )
            relationship = clamp01((world.player.relationships.get(faction_id, 0.0) + 1.0) * 0.5)
            resource_fit = self._resource_fit(world, template)

            breakdown = {
                "pressure_relevance": pressure_relevance,
                "faction_agenda": clamp01(faction.agenda_pressure * 0.7 + faction_fit * 0.3),
                "novelty": novelty,
                "style_support": style_support,
                "counterpoint": counterpoint,
                "geography": geography,
                "relationship": relationship,
                "resource_fit": resource_fit,
                "authored_priority": clamp01(template.authored_priority),
            }
            score = sum(self.config.weights.get(key, 0.0) * value for key, value in breakdown.items())
            score *= 0.7 + 0.3 * (1.0 - abs(template.base_difficulty - self._desired_difficulty(world)))
            candidates.append((template, district.id, faction_id, max(0.0, score), breakdown))
    return candidates

def choose(self, world: WorldState, style: PlaystyleTracker) -> Opportunity | None:
    candidates = self.candidates(world, style)
    if not candidates:
        return None

    rng = SplitMix64(world.root_seed).fork("opportunity", world.tick, world.resolved_opportunities)
    temperature = max(0.02, self.config.temperature)
    maximum = max(candidate[3] for candidate in candidates)
    weights = [math.exp((candidate[3] - maximum) / temperature) for candidate in candidates]
    selected = rng.weighted_choice(candidates, weights)
    template, district_id, faction_id, score, breakdown = selected

    difficulty_jitter = rng.uniform(-0.06, 0.06)
    pressure = world.player.pressure
    pressure_load = sum(pressure.as_dict().values()) / len(pressure.as_dict())
    difficulty = clamp01(template.base_difficulty + difficulty_jitter + 0.08 * pressure_load)

    instance_seed = rng.next_u64()
    return Opportunity(
        instance_id=f"opp_{stable_hash64((template.id, world.tick, instance_seed)):016x}",
        template_id=template.id,
        district_id=district_id,
        faction_id=faction_id,
        difficulty=difficulty,
        selected_tick=world.tick,
        seed=instance_seed,
        score_breakdown={**breakdown, "total": score},
    )

@staticmethod
def _pressure_relevance(world: WorldState, template: OpportunityTemplate) -> float:
    relief = 0.0
    total_weight = 0.0
    state = world.player.pressure.as_dict()
    for channel, amount in template.pressure_relief.items():
        weight = max(0.0, amount)
        relief += state.get(channel, 0.0) * weight
        total_weight += weight
    return clamp01(relief / max(1e-9, total_weight)) if total_weight else 0.25

@staticmethod
def _best_faction(world: WorldState, district_id: str) -> tuple[str, float]:
    district = world.districts[district_id]
    candidates: List[tuple[str, float]] = []
    for faction_id, faction in world.factions.items():
        control = district.faction_control.get(faction_id, 0.0)
        fit = 0.45 * control + 0.35 * faction.resources + 0.2 * faction.agenda_pressure
        candidates.append((faction_id, fit))
    return max(candidates, key=lambda pair: pair[1]) if candidates else ("", 0.0)

@staticmethod
def _resource_fit(world: WorldState, template: OpportunityTemplate) -> float:
    condition = clamp01(world.player.resources.get("condition", 1.0))
    leverage = clamp01(world.player.resources.get("leverage", 0.0))
    supports_influence = "influence" in template.resolution_methods
    supports_stealth = "stealth" in template.resolution_methods
    recovery_routes = (0.4 * leverage if supports_influence else 0.0) + (
        0.3 if supports_stealth else 0.0
    )
    return clamp01(0.35 + 0.4 * condition + recovery_routes)

@staticmethod
def _desired_difficulty(world: WorldState) -> float:
    pressure = world.player.pressure
    load = sum(pressure.as_dict().values()) / len(pressure.as_dict())
    return clamp01(0.42 + 0.18 * world.resolved_opportunities / 20.0 - 0.12 * load)
''')

def choose_method(rng: SplitMix64, strategy: Mapping[str, float], allowed: set[str]) -> str:
methods = [m for m in strategy if m in allowed]
if not methods:
methods = sorted(allowed)
weights = [strategy.get(m, 0.01) for m in methods]
return rng.weighted_choice(methods, weights)

def _method_power(world: WorldState, method: str) -> float:
base = {
"force": 0.61, "stealth": 0.59, "influence": 0.56,
"mobility": 0.58, "systems": 0.57,
}[method]
condition = world.player.resources.get("condition", 1.0)
leverage = world.player.resources.get("leverage", 0.35)
need = world.player.pressure.need
if method == "influence":
base += 0.18 * leverage
if method in ("force", "mobility"):
base += 0.12 * condition
if method in ("systems", "stealth"):
base += 0.06 * (1.0 - world.player.pressure.exposure)
base -= 0.1 * world.player.pressure.injury
base -= 0.06 * need
return clamp01(base)

def resolve_opportunity(
world: WorldState,
template: OpportunityTemplate,
method: str,
rng: SplitMix64,
tracker: PlaystyleTracker,
rumors: RumorEngine,
pressure_tuning: Mapping,
) -> Dict[str, float | bool | str]:
opportunity = world.active_opportunity
assert opportunity is not None

power = _method_power(world, method)
execution = rng.uniform(-0.16, 0.16)
success_margin = power + execution - opportunity.difficulty
success = success_margin >= 0.0
quality = clamp01(0.5 + success_margin)
visibility = clamp01({
    "force": 0.82, "stealth": 0.18, "influence": 0.35,
    "mobility": 0.66, "systems": 0.28,
}[method] + rng.uniform(-0.12, 0.12))

event = WorldEvent(
    event_id=f"evt_{stable_hash64((world.root_seed, world.tick, opportunity.instance_id)):016x}",
    tick=world.tick,
    kind="job_resolved" if success else "job_setback",
    actor_id="player",
    target_ids=[],
    district_id=opportunity.district_id,
    intensity=clamp01(0.45 + opportunity.difficulty * 0.5),
    visibility=visibility,
    method=method,
    identity_key=world.player.identity_key,
    tags={"success" if success else "setback"} | ({"protective"} if template.id == "clinic_midnight" else set()),
    metadata={"identity_ambiguity": 0.35 if method == "stealth" else 0.0, "quality": quality},
)
world.append_event(event)
tracker.record(event)
rumors.witness_event(world, event)

residue = pressure_tuning["event_method_residue"][method]
multiplier = 1.0 if success else pressure_tuning["failure_pressure_multiplier"]
for channel, amount in residue.items():
    world.player.pressure.add(
        channel,
        amount * multiplier * (0.4 + 0.6 * visibility),
        district_id=event.district_id,
        identity_key=event.identity_key,
    )
for channel, amount in template.pressure_generation.items():
    world.player.pressure.add(
        channel,
        amount * multiplier * (0.25 + 0.75 * visibility),
        district_id=event.district_id,
        identity_key=event.identity_key,
    )
if success:
    for channel, amount in template.pressure_relief.items():
        world.player.pressure.add(
            channel,
            -amount * pressure_tuning["success_relief_multiplier"],
            district_id=event.district_id,
            identity_key=event.identity_key,
        )

# Bounded resource consequences.
if method == "influence":
    world.player.resources["leverage"] = clamp01(world.player.resources.get("leverage", 0.35) - 0.06)
elif success:
    world.player.resources["leverage"] = clamp01(world.player.resources.get("leverage", 0.35) + 0.025)
if not success:
    world.player.resources["condition"] = clamp01(world.player.resources.get("condition", 1.0) - 0.06)
else:
    world.player.resources["money"] = clamp01(world.player.resources.get("money", 0.5) + 0.035)

faction = world.factions[opportunity.faction_id]
if success:
    faction.respect = clamp01(faction.respect + 0.035)
    faction.trust = clamp01(faction.trust + (0.02 if method != "force" else 0.005))
else:
    faction.awareness = clamp01(faction.awareness + 0.035)
    faction.fear = clamp01(faction.fear + (0.025 if method == "force" else 0.008))

world.player.district_id = opportunity.district_id
world.recent_template_ticks[template.id] = world.tick
world.resolved_opportunities += 1
return {
    "success": success, "quality": quality, "visibility": visibility,
    "method": method, "difficulty": opportunity.difficulty,
}
def run_campaign(
seed: int,
strategy_name: str = "hybrid",
turns: int = 36,
*,
return_trace: bool = False,
) -> Dict:
if strategy_name not in STRATEGIES:
raise KeyError(f"unknown strategy: {strategy_name}")
world = build_sample_world(seed)
templates = build_templates()
templates_by_id = {t.id: t for t in templates}
director = OpportunityDirector(templates)
tracker = PlaystyleTracker()
rumors = RumorEngine()
root_rng = SplitMix64(seed)
pressure_tuning = {
"event_method_residue": {
"force": {"exposure": 0.14, "heat": 0.16, "injury": 0.08, "need": 0.08},
"stealth": {"exposure": 0.035, "heat": 0.02, "need": 0.04},
"influence": {"debt": 0.11, "exposure": 0.025, "need": 0.02},
"mobility": {"heat": 0.09, "exposure": 0.07, "injury": 0.04},
"systems": {"exposure": 0.045, "anomaly": 0.04, "need": 0.035},
},
"failure_pressure_multiplier": 1.35,
"success_relief_multiplier": 0.22,
}
decay_rates = {
"exposure": 0.006, "heat": 0.01, "need": 0.002,
"injury": 0.0, "debt": 0.0, "anomaly": 0.001, "volatility": 0.001,
}
successes = 0
trace: List[Dict] = []

for turn in range(turns):
    world.tick += 1
    world.player.pressure.decay(decay_rates)
    if turn % 3 == 0:
        rumors.propagate(world)
    if turn % 5 == 0:
        rumors.decay(world)

    opportunity = director.choose(world, tracker)
    if opportunity is None:
        world.tick += 1
        continue
    world.active_opportunity = opportunity
    template = templates_by_id[opportunity.template_id]
    rng = root_rng.fork("resolve", turn, opportunity.instance_id)
    method = choose_method(rng, STRATEGIES[strategy_name], template.resolution_methods)
    result = resolve_opportunity(
        world, template, method, rng, tracker, rumors, pressure_tuning
    )
    successes += int(bool(result["success"]))
    if return_trace:
        trace.append({
            "turn": turn,
            "opportunity": opportunity.template_id,
            "district": opportunity.district_id,
            "faction": opportunity.faction_id,
            **result,
            "pressure": world.player.pressure.as_dict(),
            "style": tracker.normalized(),
        })

pressure = world.player.pressure.as_dict()
total_claims = sum(len(npc.knowledge) for npc in world.npcs.values())
metrics = {
    "seed": seed,
    "strategy": strategy_name,
    "turns": turns,
    "success_rate": successes / max(1, turns),
    "money": world.player.resources.get("money", 0.0),
    "leverage": world.player.resources.get("leverage", 0.0),
    "condition": world.player.resources.get("condition", 0.0),
    "pressure": pressure,
    "pressure_load": sum(pressure.values()) / len(pressure),
    "style": tracker.normalized(),
    "style_entropy": tracker.entropy(),
    "claims": total_claims,
    "events": len(world.events),
    "resolved": world.resolved_opportunities,
}
if return_trace:
    metrics["trace"] = trace
return metrics
def run_batch(runs: int = 100, turns: int = 36) -> Dict[str, Dict]:
if runs <= 0 or turns <= 0:
raise ValueError("runs and turns must be positive")
output: Dict[str, Dict] = {}
for strategy in STRATEGIES:
rows = [run_campaign(seed=1000 + i, strategy_name=strategy, turns=turns) for i in range(runs)]
keys = ["success_rate", "money", "leverage", "condition", "pressure_load", "style_entropy", "claims"]
summary = {
key: sum(float(row[key]) for row in rows) / len(rows)
for key in keys
}
summary["runs"] = runs
summary["turns"] = turns
output[strategy] = summary

# Relative dominance diagnostic.
success_mean = sum(v["success_rate"] for v in output.values()) / len(output)
pressure_mean = sum(v["pressure_load"] for v in output.values()) / len(output)
reward_mean = sum(v["money"] for v in output.values()) / len(output)
for strategy, summary in output.items():
    summary["dominance_proxy"] = (
        (summary["success_rate"] - success_mean)
        + (summary["money"] - reward_mean)
        - (summary["pressure_load"] - pressure_mean)
    )
return output
''')

from future import annotations

import argparse
import json

from .simulation import run_batch, run_campaign

def main() -> int:
parser = argparse.ArgumentParser(description="Nightglass reference simulation")
sub = parser.add_subparsers(dest="command", required=True)

one = sub.add_parser("campaign")
one.add_argument("--seed", type=int, default=1)
one.add_argument("--strategy", default="hybrid")
one.add_argument("--turns", type=int, default=36)
one.add_argument("--trace", action="store_true")

    result = run_campaign(args.seed, args.strategy, args.turns, return_trace=args.trace)
else:
    result = run_batch(args.runs, args.turns)
print(json.dumps(result, indent=2, sort_keys=True))
return 0
if name == "main":
raise SystemExit(main())

rel = path.relative_to(root)
except ValueError:
continue
yield path, rel

def detect_engine(root: Path) -> Dict:
evidence = []
engine = "unknown"
version = ""
unity_version = root / "ProjectSettings" / "ProjectVersion.txt"
if unity_version.exists():
engine = "unity"
evidence.append(str(unity_version.relative_to(root)))
text = unity_version.read_text(errors="replace")
match = re.search(r"m_EditorVersion:\s*(.+)", text)
version = match.group(1).strip() if match else ""
godot = root / "project.godot"
if godot.exists():
engine = "godot"
evidence.append("project.godot")
text = godot.read_text(errors="replace")
match = re.search(r'config/features=PackedStringArray("([^"]+)', text)
version = match.group(1) if match else ""
uprojs = list(root.glob("*.uproject"))
if uprojs:
engine = "unreal"
evidence.extend(str(p.relative_to(root)) for p in uprojs)
try:
data = json.loads(uprojs[0].read_text(errors="replace"))
version = str(data.get("EngineAssociation", ""))
except Exception:
pass
return {"engine": engine, "version": version, "evidence": evidence}

def safe_text(path: Path, limit: int = 1_000_000) -> str:
try:
if path.stat().st_size > limit:
return ""
return path.read_text(encoding="utf-8", errors="replace")
except OSError:
return ""

def sha256(path: Path) -> str:
digest = hashlib.sha256()
try:
with path.open("rb") as handle:
for block in iter(lambda: handle.read(1024 * 1024), b""):
digest.update(block)
return digest.hexdigest()
except OSError:
return ""

def probe(root: Path) -> Dict:
engine = detect_engine(root)
ext_counts = collections.Counter()
ext_bytes = collections.Counter()
group_counts = collections.Counter()
group_bytes = collections.Counter()
large_files = []
keyword_hits = {key: [] for key in KEYWORD_GROUPS}
duplicate_candidates: Dict[Tuple[int, str], List[str]] = collections.defaultdict(list)
source_sizes = []
files_total = 0
bytes_total = 0

for path, rel in iter_files(root):
    files_total += 1
    try:
        size = path.stat().st_size
    except OSError:
        continue
    bytes_total += size
    ext = path.suffix.lower() or "<none>"
    ext_counts[ext] += 1
    ext_bytes[ext] += size
    for group, extensions in ASSET_GROUPS.items():
        if ext in extensions:
            group_counts[group] += 1
            group_bytes[group] += size
    if size >= 25 * 1024 * 1024:
        large_files.append({"path": str(rel), "bytes": size})
    if ext in ASSET_GROUPS["scripts"]:
        source_sizes.append({"path": str(rel), "bytes": size, "lines": safe_text(path).count("\n") + 1})
    if ext in TEXT_EXTS:
        text = safe_text(path).lower()
        haystack = (str(rel).lower() + "\n" + text)
        for group, words in KEYWORD_GROUPS.items():
            if any(word in haystack for word in words):
                if len(keyword_hits[group]) < 40:
                    keyword_hits[group].append(str(rel))
    if 0 < size <= 128 * 1024 * 1024:
        digest = sha256(path)
        if digest:
            duplicate_candidates[(size, digest)].append(str(rel))

duplicates = [
    {"bytes_each": size, "count": len(paths), "paths": paths[:20]}
    for (size, _digest), paths in duplicate_candidates.items()
    if len(paths) > 1 and size > 4096
]
duplicates.sort(key=lambda x: x["bytes_each"] * (x["count"] - 1), reverse=True)
large_files.sort(key=lambda x: x["bytes"], reverse=True)
source_sizes.sort(key=lambda x: (x["lines"], x["bytes"]), reverse=True)

markers = {
    "git": (root / ".git").exists(),
    "github_actions": (root / ".github" / "workflows").exists(),
    "license_files": [p.name for p in root.glob("LICENSE*")],
    "readme_files": [p.name for p in root.glob("README*")],
}

risks = []
if engine["engine"] == "unknown":
    risks.append("Engine could not be identified from standard project markers.")
if not markers["git"]:
    risks.append("No .git directory detected at the supplied root.")
if not keyword_hits["test"]:
    risks.append("No obvious automated test signals were found.")
if not keyword_hits["save"]:
    risks.append("No obvious save/serialization owner was found; locate it before adding persistent simulation.")
if len(keyword_hits["event"]) == 0:
    risks.append("No obvious event/signal seam was found; avoid introducing a global bus before architecture review.")
if duplicates:
    reclaim = sum(d["bytes_each"] * (d["count"] - 1) for d in duplicates)
    if reclaim > 100 * 1024 * 1024:
        risks.append(f"Potential duplicate assets exceed {reclaim / (1024**2):.1f} MiB; verify imports before deletion.")

return {
    "root": str(root),
    "engine": engine,
    "summary": {"files": files_total, "bytes": bytes_total},
    "extensions": [
        {"extension": ext, "count": ext_counts[ext], "bytes": ext_bytes[ext]}
        for ext in sorted(ext_counts, key=lambda e: ext_bytes[e], reverse=True)
    ],
    "asset_groups": {
        group: {"count": group_counts[group], "bytes": group_bytes[group]}
        for group in ASSET_GROUPS
    },
    "keyword_hits": keyword_hits,
    "largest_files": large_files[:100],
    "largest_sources": source_sizes[:100],
    "duplicates": duplicates[:100],
    "markers": markers,
    "risks": risks,
}

write("tools/validate_content.py", r'''
#!/usr/bin/env python3
from future import annotations

import argparse
import json
import re
import sys
from pathlib import Path
from typing import Any, Dict, Iterable, List, Set

ID_PATTERN = re.compile(r"^[a-z0-9_-]+$")

FILES = {
"factions.json": ("faction", {"id", "display_name", "values", "needs", "red_lines", "traits", "starting_state"}),
"districts.json": ("district", {"id", "display_name", "security", "prosperity", "volatility", "tags", "faction_control"}),
"npc_archetypes.json": ("npc", {"id", "display_name", "role_tags", "traits", "senses", "goals"}),
"abilities.json": ("ability", {"id", "display_name", "method_weights", "costs", "residue", "tags"}),
"opportunities.json": ("opportunity", {
"id", "display_name", "family", "premise", "required_tags", "forbidden_tags",
"role_slots", "objective_nodes", "resolution_methods", "fail_forward",
"pressure_relief", "pressure_generation", "novelty_tags", "base_difficulty"
}),
}

def load_json(path: Path) -> Any:
try:
return json.loads(path.read_text(encoding="utf-8"))
except Exception as exc:
raise ValueError(f"{path}: invalid JSON: {exc}") from exc

def validate_range(errors: List[str], path: str, value: Any, lo: float = 0.0, hi: float = 1.0) -> None:
if not isinstance(value, (int, float)) or isinstance(value, bool) or not lo <= float(value) <= hi:
errors.append(f"{path}: expected number in [{lo}, {hi}], got {value!r}")

def validate(content: Path) -> List[str]:
errors: List[str] = []
ids: Dict[str, Set[str]] = {}

for filename, (kind, required) in FILES.items():
    path = content / filename
    if not path.exists():
        errors.append(f"{path}: missing")
        continue
    data = load_json(path)
    if not isinstance(data, list):
        errors.append(f"{path}: expected array")
        continue
    seen: Set[str] = set()
    for index, item in enumerate(data):
        prefix = f"{path}[{index}]"
        if not isinstance(item, dict):
            errors.append(f"{prefix}: expected object")
            continue
        missing = sorted(required - set(item))
        if missing:
            errors.append(f"{prefix}: missing keys {missing}")
        item_id = item.get("id")
        if not isinstance(item_id, str) or not ID_PATTERN.match(item_id):
            errors.append(f"{prefix}.id: invalid stable id {item_id!r}")
        elif item_id in seen:
            errors.append(f"{prefix}.id: duplicate {item_id!r}")
        else:
            seen.add(item_id)

        for key in ("security", "prosperity", "volatility", "base_difficulty", "authored_priority"):
            if key in item:
                validate_range(errors, f"{prefix}.{key}", item[key])
        for map_key in ("traits", "method_weights", "faction_control"):
            if map_key in item and isinstance(item[map_key], dict):
                for key, value in item[map_key].items():
                    validate_range(errors, f"{prefix}.{map_key}.{key}", value)
    ids[kind] = seen

# Cross references.
districts_path = content / "districts.json"
if districts_path.exists():
    for i, district in enumerate(load_json(districts_path)):
        for faction_id in district.get("faction_control", {}):
            if faction_id not in ids.get("faction", set()):
                errors.append(f"{districts_path}[{i}].faction_control: unknown faction {faction_id!r}")

opportunities_path = content / "opportunities.json"
if opportunities_path.exists():
    allowed_methods = {"force", "stealth", "influence", "mobility", "systems"}
    for i, opportunity in enumerate(load_json(opportunities_path)):
        methods = set(opportunity.get("resolution_methods", []))
        unknown = methods - allowed_methods
        if unknown:
            errors.append(f"{opportunities_path}[{i}].resolution_methods: unknown {sorted(unknown)}")
        if len(methods) < 2:
            errors.append(f"{opportunities_path}[{i}]: reusable opportunity needs at least two methods")
        node_ids = {node.get("id") for node in opportunity.get("objective_nodes", [])}
        for node in opportunity.get("objective_nodes", []):
            for nxt in node.get("next", []):
                if nxt not in node_ids:
                    errors.append(f"{opportunities_path}[{i}]: node {node.get('id')} references missing {nxt}")
        if not opportunity.get("fail_forward"):
            errors.append(f"{opportunities_path}[{i}]: missing fail-forward outcomes")

return errors
def main() -> int:
parser = argparse.ArgumentParser(description="Validate Nightglass content")
parser.add_argument("content", type=Path, nargs="?", default=Path("content"))
args = parser.parse_args()
errors = validate(args.content)
if errors:
print("Content validation failed:")
for error in errors:
print(f" - {error}")
return 1
print("Content validation passed.")
return 0

if name == "main":
raise SystemExit(main())
''')

write("shaders/NG_Common.hlsl", r'''
#ifndef NG_COMMON_INCLUDED
#define NG_COMMON_INCLUDED

#ifndef NG_SATURATE
#define NG_SATURATE(x) saturate(x)
#endif

static const float NG_EPSILON = 1e-5;
static const float NG_PI = 3.14159265359;

float NG_SafePow(float value, float exponent)
{
return pow(max(value, NG_EPSILON), exponent);
}

float3 NG_SafeNormalize(float3 value)
{
return value * rsqrt(max(dot(value, value), NG_EPSILON));
}

float NG_Hash11(float p)
{
// Stable scalar hash for authored flicker/noise. Feed quantized time, not frame index.
p = frac(p * 0.1031);
p *= p + 33.33;
p *= p + p;
return frac(p);
}

float NG_TriNoise(float t, float seed)
{
float a = NG_Hash11(floor(t) + seed);
float b = NG_Hash11(floor(t) + 1.0 + seed);
float f = frac(t);
f = f * f * (3.0 - 2.0 * f);
return lerp(a, b, f);
}

float3 NG_Overlay(float3 baseColor, float3 blendColor)
{
float3 lo = 2.0 * baseColor * blendColor;
float3 hi = 1.0 - 2.0 * (1.0 - baseColor) * (1.0 - blendColor);
return lerp(lo, hi, step(0.5, baseColor));
}

#endif
''')

write("shaders/NG_DecalBlend.hlsl", r'''
#ifndef NG_DECAL_BLEND_INCLUDED
#define NG_DECAL_BLEND_INCLUDED

#include "NG_Common.hlsl"

// Reoriented normal mapping. Inputs are tangent-space normals in [-1, 1].
float3 NG_BlendNormalRNM(float3 baseNormal, float3 detailNormal)
{
float3 t = baseNormal + float3(0.0, 0.0, 1.0);
float3 u = detailNormal * float3(-1.0, -1.0, 1.0);
return NG_SafeNormalize(t * dot(t, u) / max(t.z, NG_EPSILON) - u);
}

struct NG_MaterialLayer
{
float3 baseColor;
float3 normalTS;
float roughness;
float metallic;
float ao;
float emissiveMask;
};

NG_MaterialLayer NG_BlendMaterialLayer(
NG_MaterialLayer baseLayer,
NG_MaterialLayer overlayLayer,
float opacity,
float normalStrength)
{
float a = NG_SATURATE(opacity);
NG_MaterialLayer result;
result.baseColor = lerp(baseLayer.baseColor, overlayLayer.baseColor, a);
float3 overlayNormal = NG_SafeNormalize(lerp(float3(0.0, 0.0, 1.0), overlayLayer.normalTS, NG_SATURATE(normalStrength)));
result.normalTS = NG_SafeNormalize(lerp(baseLayer.normalTS, NG_BlendNormalRNM(baseLayer.normalTS, overlayNormal), a));
result.roughness = lerp(baseLayer.roughness, overlayLayer.roughness, a);
result.metallic = lerp(baseLayer.metallic, overlayLayer.metallic, a);
result.ao = lerp(baseLayer.ao, overlayLayer.ao, a);
result.emissiveMask = max(baseLayer.emissiveMask, overlayLayer.emissiveMask * a);
return result;
}

#endif
''')

write("shaders/NG_WetSurface.hlsl", r'''
#ifndef NG_WET_SURFACE_INCLUDED
#define NG_WET_SURFACE_INCLUDED

#include "NG_Common.hlsl"
#include "NG_DecalBlend.hlsl"

struct NG_WetSurfaceParams
{
float wetness; // 0..1 global/local blend
float puddleMask; // authored/cavity mask
float upwardnessPower; // 1..16
float roughnessMin; // material-specific wet minimum
float roughnessCompression; // 0..1
float normalFlattening; // 0..1
float darkening; // generally 0..0.25
float clearcoatBoost; // optional engine mapping
float rippleStrength; // pooled regions only
};

struct NG_WetSurfaceResult
{
float3 baseColor;
float3 normalTS;
float roughness;
float clearcoat;
float pooledWater;
};

float NG_ComputePooledWater(
float3 worldNormal,
float cavityOrAuthoredMask,
float localWetness,
float upwardnessPower)
{
float up = NG_SafePow(NG_SATURATE(worldNormal.y), max(1.0, upwardnessPower));
return NG_SATURATE(localWetness * cavityOrAuthoredMask * up);
}

NG_WetSurfaceResult NG_ApplyWetSurface(
float3 baseColor,
float3 normalTS,
float roughness,
float3 worldNormal,
float3 rippleNormalTS,
NG_WetSurfaceParams p)
{
NG_WetSurfaceResult result;
float wet = NG_SATURATE(p.wetness);
float pool = NG_ComputePooledWater(worldNormal, p.puddleMask, wet, p.upwardnessPower);
float film = NG_SATURATE(wet * 0.65 + pool * 0.55);

// Wet materials darken modestly. Do not crush albedo or tint everything blue.
result.baseColor = baseColor * (1.0 - NG_SATURATE(p.darkening) * film);

float compressed = lerp(roughness, max(p.roughnessMin, roughness * (1.0 - p.roughnessCompression)), film);
result.roughness = lerp(compressed, p.roughnessMin, pool);

float3 flattened = NG_SafeNormalize(lerp(normalTS, float3(0.0, 0.0, 1.0), NG_SATURATE(p.normalFlattening * film)));
float3 ripple = NG_SafeNormalize(lerp(float3(0.0, 0.0, 1.0), rippleNormalTS, NG_SATURATE(p.rippleStrength * pool)));
result.normalTS = NG_BlendNormalRNM(flattened, ripple);
result.clearcoat = NG_SATURATE(p.clearcoatBoost * film);
result.pooledWater = pool;
return result;
}

#endif
''')

def write_png(path: Path, width: int, height: int, rows: list[bytes], color_type: int = 0) -> None:
"""Write 8-bit PNG. color_type 0=grayscale, 6=RGBA."""
path.parent.mkdir(parents=True, exist_ok=True)
signature = b"\x89PNG\r\n\x1a\n"
def chunk(kind: bytes, data: bytes) -> bytes:
return struct.pack(">I", len(data)) + kind + data + struct.pack(">I", zlib.crc32(kind + data) & 0xffffffff)
ihdr = struct.pack(">IIBBBBB", width, height, 8, color_type, 0, 0, 0)
raw = b"".join(b"\x00" + row for row in rows)
payload = signature + chunk(b"IHDR", ihdr) + chunk(b"IDAT", zlib.compress(raw, 9)) + chunk(b"IEND", b"")
path.write_bytes(payload)

def bayer(n: int) -> list[list[int]]:
if n == 1:
return [[0]]
prev = bayer(n // 2)
out = [[0] * n for _ in range(n)]
for y in range(n // 2):
for x in range(n // 2):
v = prev[y][x] * 4
out[y][x] = v
out[y][x + n // 2] = v + 2
out[y + n // 2][x] = v + 3
out[y + n // 2][x + n // 2] = v + 1
return out

cell = 64
atlas = 4
W = H = cell * atlas

def synth_wav(path: Path, kind: str, duration: float = 0.5, rate: int = 48000) -> None:
path.parent.mkdir(parents=True, exist_ok=True)
frames = []
for i in range(int(duration * rate)):
t = i / rate
attack = min(1.0, t / 0.008)
release = max(0.0, 1.0 - t / duration) ** 2.4
env = attack * release
if kind == "pressure":
f = 180 + 210 * (t / duration) ** 1.5
sample = math.sin(2 * math.pi * f * t) + 0.35 * math.sin(2 * math.pi * (f * 1.5) * t)
elif kind == "rumor":
f = 780 - 220 * t / duration
sample = math.sin(2 * math.pi * f * t) * (0.7 + 0.3 * math.sin(2 * math.pi * 7 * t))
sample += 0.22 * math.sin(2 * math.pi * (f * 0.5) * t)
else:
f1 = 320 + 80 * t / duration
f2 = 480 + 120 * t / duration
sample = math.sin(2 * math.pi * f1 * t) + 0.6 * math.sin(2 * math.pi * f2 * t)
sample *= env * 0.18
frames.append(struct.pack("<h", int(max(-1, min(1, sample)) * 32767)))
with wave.open(str(path), "wb") as wav:
wav.setnchannels(1)
wav.setsampwidth(2)
wav.setframerate(rate)
wav.writeframes(b"".join(frames))
